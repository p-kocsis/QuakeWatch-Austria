JsonData
service in module resources

Description

REST Interface

Diese Factory wird in der App verwendet um Erdbebendaten zu laden. Es ist ein "Interface" welches den einfachen austausch der Endpunkte ermöglicht Für nähere Informationen zur implementierung einer eigenen Factory bei der Dokumentation für resources nachschlagen

Methods

  • AutPromise()

    Wenn dieses Attribut die Aut Erdbebendaten zurückliefert dann wird die Applikation gestartet. -> Start der Applikation hängt von diesem Attribut ab

    Example

    var autData = JsonData.AutPromise;

  • getAut()

    Österreichische Erdbebendaten abfragen

    Returns
    [quakeData]

    Österreichische Erbebendaten

    Example

    JsonData.getAut()

  • getEu()

    Europäische Erdbebendaten abfragen

    Returns
    [quakeData]

    Erdbebendaten nach EU gefiltert(Kontinent)

    Example

    JsonData.getEu()

  • getQuakefromIdWorld()

    Ein erdbeben nach id Abfragen wird bei der erdbeben detail Seite benötigt

    Returns
    quakeData

    Erdbeben der ID entprechend

    Example

    JsonData.getAut()

  • getWorld()

    Österreichische Erdbebendaten abfragen

    Returns
    [quakeData]

    Alle Erdbebendaten inkl. Österrreichische

    Example

    JsonData.getWorld()

  • isOnline()

    Den Online-Status der App überprüfen

    Returns
    boolean

    Liefert den Online-Status der App

    Example

    JsonData.isOnline()

  • reloadData(Erdbebendaten)

    Mithilfe von dieser funktion werden alle Erdbebendaten neu geladen

    Parameters
    ParamTypeDetails
    Erdbebendaten[quakeData]

    entsprechend der Lokation

    Example

    JsonData.reloadData('eu');

  • setOnline(online)

    Diese Funktion wird in der app.js aufgerufen um den Online-Status der App zu setzten.

    Parameters
    ParamTypeDetails
    onlineboolean

    status der App hängt von der AutPromise ab

    Example

    JsonData.setOnline(online)
